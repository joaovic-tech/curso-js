# Principios SOLID

## SRP (Single Responsibility Principle) - Princípio da Responsabilidade Única

Basicamente diz que uma classe deve ter apenas uma responsabilidade.

Por exemplo, uma classe que é responsável por gerenciar o carrinho de compras não deve ser responsável por enviar o email de confirmação de compra.

Então, para resolver isso, devemos criar uma classe responsável apenas por enviar o email de confirmação de compra.

Isso ajuda a manter o código mais limpo e fácil de manter.
